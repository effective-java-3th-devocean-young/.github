### 📅 Study 일정

### 🙌 Study 방법

2주에 1개의 Chapter를 스터디 해오는 것이 목표.

스터디 한 내용은 GitHub Repo에 PR을 하여 Code Base로 문서를 관리.

Meet Up 때는 각 Chapter의 Item을 5개의 Part로 나누어 랜덤 발표 진행.

Code Base로 관리된 문서는 GitHub Page에 자동으로 배포 될 수 있도록 DevOps 작업 진행.
